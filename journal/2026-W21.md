The contrast with this thesis's z-inversion-into-two-framework-instances is *mechanistic* (partition returns by classification vs partition insertions by sign), not topological. The "no published work uses z-inversion for ceiling mapping" framing survives; the language in 01 needed to be sharpened to name the partition mechanism explicitly instead of describing Buchanan as "separate `grid_map` instances" (he isn't).

- *Miki 2022* — arXiv PDF (25 MB; direct curl worked where WebFetch had failed yesterday). §II-C is more nuanced than the lit-review "maximum-height kernel" shorthand: the framework runs a one-dimensional Kalman fusion (eqn 1) with a count-thresholded suppression rule that drops lower returns when many points fall in the same cell within one scan. Behaviour approaches max-height under dense sampling but is not literally a max kernel. The variance update (eqn 2) is the standard recursive Kalman variance plus a constant time-variance increment `σ²_t` per cycle for un-updated cells, which is the operational mechanism by which stale cells return to the outlier-rejection envelope. Independently corroborates ADR 0005: visibility cleanup (eqn 3, `if pzi < hi − σi, remove`) is *designed* to remove cells below the ray-height envelope, so applied to negated-z ceiling returns it would systematically erase the lowest overhead surface. Worth noting that Miki's framework explicitly treats overhead structure as nuisance (§II-C "exclusion area", Fig. 3) — the framework is not just silent on ceilings, it is actively designed to suppress them, which makes the contribution claim of the dual-instance reuse stronger.

- *Fankhauser 2018* — fetched via Wayback Machine snapshot of the ETH Research Collection bitstream URL after direct fetch returned the SPA HTML stub (recipe in surfaced section). 9 pages, full primary source. The variance decomposition is exactly what 02 assumed: measurement variance `σ_p² = J_S Σ_S J_S^T + J_Φ Σ_ΦIS J_Φ^T` (eqn 5; sensor-noise + sensor-rotation contributions, sensor-position uncertainty deliberately excluded by the robot-centric frame choice), Kalman update `σ_h^{2+} = σ_h^{2-} σ_p² / (σ_h^{2-} + σ_p²)` (eqn 6), and per-cell spatial covariance Σ_P_i accumulated through robot motion in eqns 14, 20–21. Critically: the full drift-aware fusion (h_min, h_max in §III-D) is a *map-fusion-step* construction, *not* exposed by the framework's runtime `variance` layer. The runtime layer publishes the per-cell Kalman variance only. This is the load-bearing distinction for what 02's δ_cal is calibrated to absorb.

- *`enable_visibility_cleanup`* — kernel-gated at `custom_kernels.py:204`. The cleanup conditional `nmap_h > nz + correction` can fire on the ceiling instance in a specific failure mode: when a ray to a *higher* new ceiling measurement passes through a cell whose previous estimate was a *lower* ceiling (less negative `nmap_h`), the lower-ceiling cell gets cleared. - *`enable_drift_compensation`* — gated by a three-way conjunction at `elevation_mapping.py:404`: flag AND `error_cnt > min_height_drift_cnt` AND significant pose noise. The middle condition is the kernel-level gate: `error_counting_kernel:336` only accumulates `error_cnt` when `map_t > traversability_inlier`, and the ceiling instance never populates `map_t` (no ceiling-trained CNN). Drift compensation never fires regardless of the flag. The `false` setting is correct but **defensively redundant** — the actual gate is the absent traversability mask, not the flag. Either setting behaves identically at runtime; `false` is preferable as a static signal that the feature is intentionally inactive.

- *`enable_overlap_clearance`* — gated at `elevation_mapping.py:433`, with `clear_overlap_map(t)` clearing cells outside `[t[2] − overlap_clear_range_z, t[2] + overlap_clear_range_z]`. Enabling overlap clearance on the ceiling would wipe the entire ceiling map every clearance pass. **Architecturally necessary `false`, not defensive** — the previous audit characterisation ("single-floor assumption") missed how catastrophically the heuristic fails on the negated-z layer.
